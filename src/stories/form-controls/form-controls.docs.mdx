# FormControls

FormControls is a set of form primitives: Input, TextArea, Select, Date, Checkbox, Radio, File, Toggle, and Stepper. Use them to build forms with consistent styling, labels, validation messages, and optional fluid layout. Access via `FormControls.Input`, `FormControls.Select`, etc.

## Features

- **Input:** Text input with label, placeholder, required indicator, error message, isFluid. `type="number"` is mapped to `<input type="text" inputmode="numeric" pattern="[0-9]*">` for predictable behavior on desktop and a numeric keypad on mobile (avoids the UX issues of native `type="number"`). For phone fields, use `autoComplete="tel"`, `phoneDigits={10}`, and `maxLength={10}` so browser autofill with a country code (e.g. `+91 98765 43210`) is normalized to the last 10 digits. `type="search"` adds a leading `search` icon, hides the browser's native clear button, and renders a custom `close` clear control that appears only when the field has content. Pass `prefix` / `suffix` (â‰¤ 4 chars) for inline affixes like currency (`$`), country code (`+91`), unit (`kg`, `%`), or TLD (`.com`).
- **TextArea:** Multi-line text with same options as Input
- **Select:** Single or multi select with options `{ label, value }`; dropdown with placeholder
- **Date:** Calendar **`Date | null`**, **Cancel / OK** staging, `date-fns` + **Floating UI** (desktop **popover** ~400px max width, **â‰¤768px** bottom sheet + backdrop). **Month/year pickers** as subviews with back + **â€śSelect a month of \{yyyy\}â€ť** / **â€śSelect a year for \{MMMM\}â€ť**. Trigger **`calendar_month`** icon; **`minDate`/`maxDate`/`disabledDates`/`disabledDaysOfWeek`**, **`locale`**, **`dateFormat`**, **`weekStartsOn`**, **`clearable`**, **`readOnly`**, **`popoverPlacement`**, **`onOpen`/`onClose`**, optional **`name`** (hidden **yyyy-MM-dd**).
- **Checkbox:** Group-first API â€” pass an `options` array (single item is fine for one checkbox). Renders `<fieldset>` + `<legend>` with `value: CheckboxValue[]` + `onChange(values, e)`. Each option supports `description` and an optional `icon`. Set `variant="card"` for tile-style options. **`dataTestId`** on the group adds suffixed test ids for E2E (see **E2E / test selectors** below).
- **Radio:** Each row is a native `radio` (visually hidden) with a custom ring/dot aligned like Checkbox/Toggle; unique `id` is derived from `name` + `value` so groups never get duplicate ids. Use `fieldset`/`legend` for the group label. **`dataTestId`** on the group uses the same suffix scheme as Checkbox.
- **File:** File picker with `button` (default) and `card` variants. Both render a removable file list below the trigger; the card variant adds a dashed drop zone with drag-and-drop. Use `multiple`, `accept`, `buttonLabel`, `dropZoneText` to tailor the UX.
- **Toggle:** Accessible switch (`checkbox` + `role="switch"`) with track/thumb; `checked` / `defaultChecked` and `onChange(checked: boolean)`
- **Stepper:** Text input for form stepper flows (label, placeholder, error)
- **Common:** label, isDisabled, isRequired, isFluid, className, error for most controls

## Usage

```jsx
import { FormControls } from "cleanplate";

const App = () => (
  <>
    <FormControls.Input
      label="Email"
      placeholder="user@example.com"
      isRequired
    />
    <FormControls.Select
      label="Fruit"
      placeholder="Select a fruit"
      options={[
        { label: "Apple", value: "apple" },
        { label: "Mango", value: "mango" },
      ]}
    />
  </>
);
```

## Examples

### Input

```jsx
<FormControls.Input placeholder="Email" />
<FormControls.Input label="Email" placeholder="user@acme.com" isRequired />
```

Phone field with browser autofill normalization (`phoneDigits` keeps the last 10 digits after country code is stripped):

```jsx
<FormControls.Input
  label="Mobile number"
  name="mobile"
  type="number"
  autoComplete="tel"
  prefix="+91"
  phoneDigits={10}
  maxLength={10}
  placeholder="10-digit mobile"
  isRequired
/>
```

## E2E / test selectors (Checkbox and Radio)

Pass **`dataTestId="interests"`** on the group. Checkbox and Radio share the same suffix scheme:

| Suffix | Element |
| --- | --- |
| *(root)* | `<fieldset>` |
| `-options` | Options container |
| `-option-{value}` | Option row |
| `-input-{value}` | Native input |
| `-label-{value}` | Clickable label |

`{value}` comes from each option's `value` prop (non-alphanumeric characters become `-`). Set **`dataTestId`** on an individual option to override only that option's `-input-{value}` id.

```jsx
<FormControls.Checkbox
  label="Email me about"
  name="interests"
  dataTestId="interests"
  options={[
    { label: "Newsletters", value: "newsletter" },
    { label: "Product updates", value: "product" },
  ]}
/>
```

```ts
// Playwright â€” prefer -label-{value} for clicks
await page.getByTestId("interests-label-newsletter").click();
await expect(page.getByTestId("interests-input-newsletter")).toBeChecked();

await page.getByTestId("shipping-label-express").click();
await expect(page.getByTestId("shipping-input-express")).toBeChecked();
```

## Behavior

- **Controlled / uncontrolled:** Use `value` + `onChange` for controlled fields, or `defaultValue` for uncontrolled. Select uses `value` (option object or array for multi) and `onChange(option)`.
- **Radio:** Group-first API â€” pass `options: RadioOption[]` and the component renders the entire `<fieldset>` + `<legend>` group. Use `value` + `onChange(value, e)` for controlled groups, or `defaultValue` for uncontrolled. `isRequired` shows `*` on the group label and is wired into the first enabled option for HTML5 validation. Each option may include `description` (muted helper text) and `icon` (`ReactNode` â€” Material `<Icon>`, brand logo, custom SVG). Set `variant="card"` for tile-style options with the ring in the top-right corner. For a single radio, just pass an array with one entry. **`dataTestId`** adds suffixed ids on the fieldset, options container, rows, inputs, and labels.
- **Checkbox:** Group-first API â€” pass `options: CheckboxOption[]` and the component renders the entire `<fieldset>` + `<legend>` group. Use `value` (`CheckboxValue[]`) + `onChange(values, e)` for controlled groups, or `defaultValue` for uncontrolled. `isRequired` shows `*` on the group label and sets `aria-required` on the group; native HTML5 doesn't enforce "at least one" for checkbox groups, so add custom validation at the form layer. Each option may include `description` (muted helper text) and `icon` (`ReactNode`). Set `variant="card"` for tile-style options. For a single checkbox, pass a one-element `options` array â€” `value=[]` is unchecked. **`dataTestId`** uses the same suffix scheme as Radio.
- **Toggle:** Switch control â€” use `checked` + `onChange(checked)` when controlled, or `defaultChecked` when uncontrolled. Renders `type="checkbox"` with `role="switch"`.
- **File:** `variant="button"` (default) renders a primary-brand trigger; `variant="card"` renders a dashed drop zone with click + drag-and-drop. Selection is a `File[]` exposed via `value` (controlled) / `defaultValue` (uncontrolled initial visual list) and emitted from `onChange(files, e)`. With `multiple`, picks/drops append to the list; without, the new selection replaces it. Each selected file renders below the trigger with a type-specific thumbnail icon, name, formatted size, and a remove button.
- **Select:** Options are `{ label: string, value: string | number }`. Single select passes one option to `onChange`; multi passes an array. Value syncs from prop in useEffect.
- **Date:** Returns **`Date | null`** to **`onChange`**. **Staging:** pick a day then **OK** to commit; **Cancel** or dismiss restores the last committed value. **Desktop:** portalled popover (flip/shift); **â‰¤768px:** bottom sheet + backdrop (viewport breakpoint matches Select). Tap **month** or **year** in the header for **scrollable lists**; subview titles include the current **year** / **full month name** for context. **`dateFormat`** + **`locale`** shape the field text; **`weekStartsOn`** shifts the grid. **`readOnly`** / **`isDisabled`** block opening. **`name`** adds a hidden **`yyyy-MM-dd`** input for native form posts. See Playground stories **Date Â· â€¦** for min/max, locale, validation, form submit, and mobile sheet.
- **Error:** When `error` is set, the field gets an error class and the message is shown below.
- **isFluid:** Applies full-width styling to the field wrapper.
