# CleanPlate

**A headless React UI framework** — reusable, accessible components you can style to match your brand. No opinionated theme; you bring the look and feel.

---

## Getting started

### 1. Install

```bash
npm install cleanplate
```

Or with Yarn:

```bash
yarn add cleanplate
```

### 2. Add styles

Import the reset and base styles once at your app root (e.g. in `main.jsx` or `App.jsx`):

```jsx
import "cleanplate/dist/index.css";
```

### 3. Import and use components

Everything is named exports from the `cleanplate` package. For example:

```jsx
import { Button, Typography } from "cleanplate";

function App() {
  return (
    <>
      <Typography variant="h1">Hello</Typography>
      <Button variant="solid" onClick={() => alert("Hi!")}>
        Say hi
      </Button>
    </>
  );
}
```

## Quick examples

Here are a few components to get a feel for the API. Use the **sidebar** to open full docs and playgrounds for each component.

### Button

Variants: `solid`, `outline`, `ghost`, `icon`. Sizes: `small`, `medium`.

```jsx
import { Button } from "cleanplate";

<Button variant="solid">Save</Button>
<Button variant="outline">Cancel</Button>
<Button variant="ghost">Skip</Button>
```

### Typography

Use for headings and body text. Variants: `h1`–`h6`, `p`, `span`, `small`.

```jsx
import { Typography } from "cleanplate";

<Typography variant="h1">Heading 1</Typography>
<Typography variant="h4">Heading 4</Typography>
<Typography variant="p">A paragraph of body text.</Typography>
```

### Badge

Short labels for status or tags.

```jsx
import { Badge } from "cleanplate";

<Badge label="New" />
<Badge label="Draft" variant="info" />
```

### Form controls

Inputs, selects, text areas, checkboxes, and more live under `FormControls`:

```jsx
import { FormControls } from "cleanplate";

<FormControls.Input
  label="Email"
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>

<FormControls.Select
  label="Country"
  options={[
    { label: "United States", value: "us" },
    { label: "Canada", value: "ca" },
  ]}
  value={country}
  onChange={(e) => setCountry(e.target.value)}
/>
```

### Icon

Material Symbols–style icons. Pass a `name` (e.g. from [Google Material Icons](https://fonts.google.com/icons)), and optional `size` and `color`.

```jsx
import { Icon } from "cleanplate";

<Icon name="settings" size="small" color="black" />
<Icon name="check_circle" size="medium" />
```

## LLM / AI-friendly documentation

CleanPlate ships **documentation written for AI assistants** (e.g. Cursor, Claude Code, or any LLM that reads your repo). Use it to get accurate component APIs, props, and examples when coding with AI.

- **`llms.txt`** (project root) — Index of all components with file paths, purpose, key features, types, and related components. Point your AI at this file so it can discover and open the right docs.
- **`docs/`** — One Markdown file per component (e.g. `docs/Button.md`, `docs/PageHeader.md`). Each doc includes props tables, TypeScript types, usage examples, behavior notes, and related components.

When working in **Cursor** or **Claude Code**, reference `llms.txt` or `docs/<ComponentName>.md` in your prompts so the assistant uses up-to-date CleanPlate APIs and patterns.
