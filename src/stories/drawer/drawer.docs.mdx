# Drawer

The `Drawer` component is a slide-in overlay panel for navigation, filters, settings, or detail views. On desktop it slides from a configurable edge; on mobile (viewport ≤768px) it always behaves as a **bottom sheet** (max height 90dvh).

## Features

- **Four placements:** `left`, `right`, `top`, `bottom` (desktop)
- **Mobile bottom sheet:** Automatic below 768px regardless of `placement`
- **Modal-like chrome:** Optional title, close button, footer primary (solid) / secondary (outline) / tertiary (ghost) actions
- **Smooth transitions:** Subtle fade and slide when opening and closing; reduced motion supported
- **Accessibility:** Focus trap, scroll lock, `role="dialog"`, ARIA labels — provide `title` or `ariaLabel` (dev warning if both omitted)
- **E2E friendly:** `dataTestId` with suffixed test hooks (same convention as Modal)
- **Style overrides:** `className`, `overlayClassName`, `contentClassName`, `headerClassName`, `bodyClassName`, `footerClassName`

## Size presets

| Size | Side width | Top / bottom height |
| --- | --- | --- |
| `small` | `min(280px, 100vw − 56px)` | `30dvh` |
| `medium` | `min(360px, 100vw − 56px)` | `50dvh` |
| `large` | `min(480px, 100vw − 56px)` | `70dvh` |
| `full` | `100vw` | `100dvh` |

Mobile (≤768px) overrides `size` with a bottom sheet (`max-height: 90dvh`, full width).

## Footer actions

When any of `primaryButtonLabel`, `secondaryButtonLabel`, or `tertiaryButtonLabel` is set, a footer row renders:

- **Tertiary** — `ghost` button, aligned to the start of the row
- **Secondary** — `outline` button
- **Primary** — `solid` button, aligned with secondary on the end

Footer buttons do not auto-close the drawer; call `onClose` in their handlers when needed.

```jsx
import { Drawer, Button, Typography } from "cleanplate";
import { useState } from "react";

const App = () => {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setIsOpen(true)}>Open Drawer</Button>
      <Drawer
        isOpen={isOpen}
        onClose={() => setIsOpen(false)}
        placement="right"
        title="Filters"
      >
        <Typography variant="p">Filter options go here.</Typography>
      </Drawer>
    </>
  );
};
```

## When to use Drawer vs Modal vs BottomSheet

These three components all show content on top of the page, but they optimize for different layout and interaction patterns. Pick based on **where the panel should sit**, **how much of the screen it should occupy**, and **whether you need gesture-based height control**.

### Quick pick

- **Drawer** — Side or edge panel that keeps page context visible (filters, nav, settings, item details). On viewports **≤768px**, any `placement` automatically becomes a **bottom sheet** (max **90dvh**) with the same header, dismiss, and focus behavior as desktop.
- **Modal** — Centered, attention-grabbing dialog for tasks that need full focus (forms, confirmations with rich content, multi-step flows). Best when the user should pause the underlying page.
- **BottomSheet** — Bottom-only panel where the user **drags** to resize or dismiss. Use when snap heights (30% / 60% / 90%) and a drag handle matter more than edge placement or built-in dialog chrome.

**Note:** Drawer’s mobile bottom sheet is **not** the same as `BottomSheet`. Drawer gives you dialog semantics, dismiss controls, and optional header/footer; `BottomSheet` is for drag-and-snap interactions without that chrome. Use `BottomSheet` when gestures are the primary UX; use `Drawer` when you want a consistent overlay panel across desktop and mobile.
